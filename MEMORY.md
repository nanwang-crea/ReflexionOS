## 协作规则

- [2026-04-19] 功能规划讨论 -> 先收敛目标和方案层，再决定是否细化为执行计划 -> 验收标准：优先提供高层结构、取舍和边界，而不是直接给出详细 implementation plan。
- [2026-04-19] 记忆系统设计 -> 先定义写入协议、召回策略和第一版边界，再谈完整 memory taxonomy 或复杂实现 -> 验收标准：优先回答“什么能写、什么时候写、如何召回”，而不是先扩展记忆类别。
- [2026-04-20] 记忆阶段优先级 -> 第一优先先做好当前项目下当前会话的连续记忆，跨会话记忆可以预留但不抢第一阶段范围 -> 验收标准：方案先围绕“当前会话如何不断片”收敛，不把跨会话能力当作第一阶段主目标。
- [2026-04-20] 记忆架构表达 -> 区分产品/治理层与运行时读取层，避免把复杂存储层直接暴露给模型 -> 验收标准：文档中明确 Storage/治理可以复杂，但 Runtime 读取协议必须极简，Candidate 不作为模型默认读取层。
- [2026-04-20] 记忆所有权与冲突处理 -> AI 和用户都可以操作记忆，但用户优先级更高；当前系统按单实际用户假设设计；冲突时先回查历史文档，再由用户确认，确认后以用户最新意见为准，可覆盖旧记忆或标记失效 -> 验收标准：蓝图与后续协议明确“用户最高优先级、先查证据、后由用户裁决”。

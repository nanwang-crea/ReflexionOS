# Codex-Style Agent 架构设计（Loop + Patch + Self-Debug）

## 📌 设计目标

构建一个类似 Codex / Cursor 的本地执行型 Agent，具备：

* 自动读写代码
* 基于 diff 的精确修改（Patch）
* 自动运行命令（测试 / 构建）
* 错误驱动的自修复能力（Self-Debug Loop）
* 高速迭代执行（非图结构）

---

## 🧠 核心思想

> 从「规划驱动」转向「执行驱动」

```text
小步执行 → 获取反馈 → 自动修复 → 收敛结果
```

## 2️⃣ Routing Layer（路由层）

### 职责

* 判断任务类型
* 决定执行模式

### 示例

```python
def classify_task(task: str):
    if any(x in task for x in ["代码", "修复", "文件", "bug"]):
        return "rapid_loop"
    return "planner"
```

---

## 3️⃣ Execution Layer（执行层🔥核心）

---

## Rapid Execution Loop

### 核心流程

```python
class RapidExecutionLoop:

    def run(self, task, context):

        for step in range(MAX_STEPS):

            prompt = build_prompt(context)

            action = llm(prompt)

            if action["type"] == "finish":
                return action["answer"]

            result = execute_tool(action)

            context.update(action, result)

            if result.error:
                context.inject_error(result)

        return "failed"
```

---

## 特点

* 无预规划
* 每步决策
* 强反馈驱动
* 自动修复

## 5️⃣ Action Protocol（LLM输出协议🔥）

---

### 统一 JSON Schema

```json
{
  "type": "tool_call | finish",
  "thought": "string",
  "tool": "string",
  "args": {},
  "confidence": 0.0
}
```

---

### 规则

* 必须是 JSON
* 不允许额外文本
* thought 简短（1句）

---

### finish 示例

```json
{
  "type": "finish",
  "thought": "完成任务",
  "answer": "修复完成",
  "confidence": 0.95
}
```

## 6️⃣ Prompt Layer（提示词层）

---

### 🔥 System Prompt

```text
You are an autonomous coding agent working in a local workspace.

You must:
- Solve tasks step by step using tools
- Prefer minimal edits (use apply_patch)
- Automatically fix errors
- Verify results via shell commands

Rules:
- Output MUST be valid JSON
- No explanation outside JSON
- Keep thought short
```

## 7️⃣ Self-Debug Layer（自修复机制🔥）

---

### 核心思想

```text
错误 = 下一轮输入
```

---

### 错误注入模板

```text
The previous action failed.

Error:
{error_message}

Relevant code:
{snippet}

Fix the issue using apply_patch.
Do not repeat the same action.
```

---

### 行为

| 情况    | 行为   |
| ----- | ---- |
| 语法错误  | 自动修复 |
| 测试失败  | 修改逻辑 |
| 文件不存在 | 搜索路径 |
| 命令失败  | 调整命令 |

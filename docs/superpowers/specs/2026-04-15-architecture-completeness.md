## 📋 MVP 实施计划

### 必须立即补充的4个组件:

#### 1. Patch 工具 🔴 关键

**问题:** 没有Patch工具,Agent无法进行精确的代码修改

**实施方案:**
```python
# 核心文件
backend/app/tools/patch_tool.py      # Patch工具
backend/app/tools/diff_parser.py     # Diff解析器
```

**功能要求:**
- Unified Diff 解析
- 安全的 Patch 应用
- 冲突检测
- 回滚机制

**预计工时:** 3小时

---

#### 2. 持久化存储层 🔴 基础

**问题:** 数据重启丢失,无法追溯历史

**实施方案:**
```python
# 核心文件
backend/app/storage/database.py      # 数据库连接
backend/app/storage/models.py        # ORM模型
backend/app/storage/repositories/    # 数据仓储
```

**技术选型:**
- SQLite (轻量级)
- SQLAlchemy (ORM)
- Alembic (迁移)

**预计工时:** 4小时

---

#### 3. 配置管理完善 🔴 体验

**问题:** 无法动态修改LLM配置

**实施方案:**
```python
# 核心文件
backend/app/config/settings.py       # 应用配置
backend/app/config/llm_config.py     # LLM配置
backend/app/config/validators.py     # 配置验证
```

**功能要求:**
- 动态配置修改
- 配置验证
- 配置持久化
- 前端配置界面

**预计工时:** 2小时

---

#### 4. WebSocket 后端 🔴 实时

**问题:** 无法实时推送执行状态

**实施方案:**
```python
# 核心文件
backend/app/api/websocket.py         # WebSocket端点
```

**功能要求:**
- 实时步骤推送
- 日志流式传输
- 连接管理
- 断线重连

**预计工时:** 3小时

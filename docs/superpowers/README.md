# Superpowers Docs Guide

## Active Docs

Current design references live in:

- `docs/superpowers/specs/`
- `docs/superpowers/plans/`

These files are the ones that should be read first when understanding the current architecture, boundaries, and active design direction.

## Archive Docs

Historical reference docs live in:

- `docs/superpowers/archive/specs/`
- `docs/superpowers/archive/plans/`

Archived docs are kept for context and historical traceability, but they are not the current execution or design source of truth.

## Rule Of Thumb

- If a doc explains the current system or a still-relevant target architecture, keep it in `specs/` or `plans/`.
- If a doc was only useful during a past cleanup/migration/document-sync step, move it to `archive/`.
